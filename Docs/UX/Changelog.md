### 주요 설계 결정

#### 1. 레이아웃 구조

- **3-패널 캔버스 영역**: Source Panel (30%) + Atlas Preview (40%) + Properties Panel (30%)
- **하단 패널 2단**: Sprite List (그리드 뷰) + Animation Panel
- **역할 명확화**: Source = 편집 작업, Atlas = 읽기 전용 결과 확인

#### 2. Sprite List Panel 개선

**변경 사항**: 가로 스크롤 → 세로 스크롤 그리드 뷰

- **문제**: 기존 PRD의 가로 스크롤 방식은 많은 스프라이트 탐색 시 비효율적
- **해결**: 2D 그리드 레이아웃 + 세로 스크롤 + 썸네일 + ID 라벨
- **이점**: 한 화면에 더 많은 스프라이트 표시, 직관적 탐색

#### 3. ID 검증 실시간 피드백

**추가 기능**: Properties Panel에 실시간 검증 인디케이터

- **문제**: 기존에는 Export 시점에만 중복 ID 발견
- **해결**: 입력 중 실시간 검증 + 시각적 피드백 (녹색 체크/빨간 경고)
- **이점**: 조기 에러 발견, 사용자 경험 개선

#### 4. Auto Slice 후 자동 툴 전환

**추가 흐름**: Auto Slice 완료 → Select Tool 자동 활성화

- **문제**: Auto Slice 후 수동으로 툴 변경 필요
- **해결**: 슬라이싱 완료 시 자동으로 Select Tool 활성화
- **이점**: 워크플로우 단축, 즉시 편집 가능

#### 5. 스프라이트 선택 동기화

**추가 흐름**: 3-Way 동기화

- Sprite List 선택 → Properties Panel + Source Panel Highlight + Atlas Preview Highlight
- **이점**: 현재 작업 컨텍스트 명확화

#### 6. 9-Slice 섹션 Collapsible 처리

**변경 사항**: Properties Panel 내 Collapsible Section

- **문제**: 9-Slice 설정이 패널 공간 과다 점유
- **해결**: 접을 수 있는 섹션 (▶/▼) + Preview는 별도 다이얼로그
- **이점**: 공간 효율성 증대

#### 7. Animation Editor 별도 다이얼로그 분리

**변경 사항**: Animation Panel = 목록 뷰, 편집은 별도 Dialog

- **문제**: Animation Panel에서 복잡한 편집 UI 표시 어려움
- **해결**: 목록에서 [Edit] 버튼 → 전용 Animation Editor Dialog
- **이점**: 집중된 작업 환경, 프리뷰 플레이어 제공

### 워크플로우 설계

총 **10개 주요 워크플로우** 정의:

1. 프로젝트 시작 (신규/기존)
2. 수동 슬라이싱
3. 그리드 슬라이싱
4. 자동 슬라이싱 (투명도 기반)
5. 스프라이트 선택 및 속성 편집
6. ID 편집 및 실시간 검증
7. 피봇 설정 (MVP)
8. 아틀라스 내보내기
9. 프로젝트 저장/로드
10. 애니메이션 시퀀스 편집 (Phase 3)

### 에러 처리 및 검증

| 검증 시점 | 검증 내용 | 피드백 방식 |
|----------|----------|------------|
| 이미지 로드 | PNG 포맷, 파일 유효성 | 에러 다이얼로그 |
| ID 입력 중 | 중복 검증, 빈 값 검증 | 실시간 인디케이터 |
| Export 전 | ID 중복 최종 검증 | 에러 다이얼로그 + 차단 |
| 빈 패킹 | 공간 부족 여부 | 에러 다이얼로그 + 해결 제안 |
| 프로젝트 로드 | 버전 호환성, 소스 파일 존재 | 경고 메시지 + 재선택 |
